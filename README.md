## Описание проекта

**DenoiseBuddy** — это многофункциональный асинхронный телеграм-бот, созданный для работы с изображениями. Он позволяет не только восстанавливать поврежденные участки изображений (inpainting), но и генерировать совершенно новые изображения по текстовому описанию, а также просматривать историю своих запросов.

Проект построен с использованием современного асинхронного фреймворка `aiogram 3.x`, имеет поддержку GPU-ускорений для вычислений и использует `Supabase` в качестве бэкенда для хранения данных.

## Основные возможности

*   **Восстановление изображений (Inpainting):** Использует алгоритм ADMM (Alternating Direction Method of Multipliers) для интеллектуального дозаполнения недостающих или поврежденных частей изображения.
*   **Генерация изображений:** Интегрирован с API `FusionBrain` для создания уникальных изображений на основе текстовых запросов пользователя.
*   **История запросов:** Каждый пользователь может просмотреть историю своих операций (восстановления и генерации) и получить доступ к результатам.
*   **Поддержка GPU:** Для значительно ускорения вычислений при восстановлении изображений используется библиотека `CuPy`, задействующая ресурсы NVIDIA GPU. При отсутствии GPU проект автоматически переключается на CPU (NumPy).
*   **Многоязычность:** Интерфейс бота поддерживает несколько языков благодаря встроенной системе локализации на основе `gettext`.
*   **Приятный бонус:** Секретная "магическая" кнопка для поднятия настроения! 😺

## Технический стек

*   **Язык программирования:** Python 3.12+
*   **Фреймворк для Telegram-бота:** `aiogram 3.x`
*   **Вычисления:** `NumPy` (CPU), `CuPy` (NVIDIA GPU)
*   **Бэкенд как сервис (BaaS):** `Supabase` (хранение файлов и данных в PostgreSQL)
*   **Автоматизация задач:** `doit` (`dodo.py`)
*   **Тестирование:** `pytest`
*   **Документация:** `Sphinx`
*   **Качество кода:** `pre-commit`, `black`, `isort`, `flake8`

## Установка

### Для пользователя

1.  **Предварительные требования:**
    *   Python 3.12 или выше.
    *   (Опционально, для GPU-ускорения) **NVIDIA CUDA Toolkit**. Подробности ниже.
2.  **Установка:**
    Установите пакет с помощью `pip` из собранного `.whl` файла:
    ```bash
    pip install denoise_bot-1.0.0-py3-none-any.whl
    ```
3.  **Конфигурация:**
    Создайте файл `.env` в директории, откуда будете запускать бота, и заполните его по примеру ниже.
4.  **Запуск:**
    После установки вы сможете запустить бота простой командой в терминале:
    ```bash
    denoise-bot
    ```

### Для разработчика

1.  **Клонируйте репозиторий:**
    ```bash
    git clone <адрес_вашего_репозитория>
    cd denoise_bot
    ```
2.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv .venv
    source .venv/bin/activate  # Для Linux/macOS
    # .\.venv\Scripts\activate    # Для Windows
    ```
3.  **Установите проект в редактируемом режиме со всеми зависимостями для разработки:**
    Эта команда установит как основные зависимости проекта, так и все инструменты для тестирования, линтинга и сборки документации.
    ```bash
    pip install -e .[dev]
    ```
4.  **Настройте pre-commit хуки:**
    Это позволит автоматически форматировать код перед каждым коммитом.
    ```bash
    pre-commit install
    pre-commit install --hook-type pre-push # Для проверок flake8 перед пушем
    ```
5.  **Конфигурация:**
    Создайте файл `.env` в корне проекта и заполните его.

## ⚠️ Требования к GPU (NVIDIA CUDA)

Для полноценной работы GPU-ускорения, которое значительно сокращает время обработки изображений, в вашей системе должен быть установлен **NVIDIA CUDA Toolkit**.

*   Проект использует библиотеку `cupy-cuda12x`, которая совместима с **любой версией CUDA Toolkit 12.x** (например, 12.0, 12.1, ..., 12.5).
*   Убедитесь, что у вас установлен совместимый драйвер NVIDIA.
*   Если CUDA или CuPy не будут найдены, проект автоматически переключится на использование CPU, но производительность будет значительно ниже.

## Конфигурация (`.env` файл)

Для работы бота необходимо создать файл `.env` в корневой директории проекта и указать в нем следующие переменные:

```env
# Токен вашего телеграм-бота от @BotFather
TELEGRAM_BOT_TOKEN="ВАШ_ТОКЕН"

# Данные для подключения к вашему проекту Supabase
SUPABASE_URL="https://xxx.supabase.co"
SUPABASE_KEY="ВАШ_ANON_KEY"

# Ключи для API FusionBrain (генерация изображений)
FUSIONBRAIN_API_KEY="ВАШ_API_КЛЮЧ"
FUSIONBRAIN_API_SECRET="ВАШ_СЕКРЕТНЫЙ_КЛЮЧ"
```

## Использование (для разработчиков)

Проект использует `doit` для автоматизации рутинных задач. Файл `dodo.py` определяет все доступные команды.

*   **Показать список всех команд:**
    ```bash
    doit list
    ```
*   **Запустить тесты:**
    ```bash
    doit test
    ```
*   **Собрать HTML-документацию (результат в `docs/build/html`):**
    ```bash
    doit docs
    ```
*   **Полный цикл обновления файлов локализации:**
    (извлечь строки из кода, обновить `.po` файлы и скомпилировать `.mo` файлы)
    ```bash
    doit i18n
    ```
